`qivryn` is an open-source, modular coding agent for the command line.

It provides a battle-tested agent loop so you can simply plug in your model, rules, and tools.

![qivryn](/images/qivryn-demo.gif)

## Quick Start

<CLIInstall />

Then start using it:

```bash
# Interactive mode
qivryn

# Headless mode
qivryn -p "Generate a conventional commit name for the current git changes"
```

## How to Use Qivryn CLI - Basic Usage

Out of the box, `qivryn` comes with tools that let it understand your codebase, edit files, run terminal commands, and more (if you approve). You can ask `qivryn` to:

- Fix failing tests
- Find something in the codebase
- Execute a refactor
- Write a new feature
- And a lot more

Use '@' to give it file context, or '/' to run slash commands.

If you want to resume a previous conversation, run `qivryn --resume`.

## How to Use Headless Mode (`-p` flag)

In headless mode, `qivryn` will only output its final response, making it perfect for Unix Philosophy-style scripting and automation. For example, you could pipe your git diff into `qivryn` to generate a commit message, and write this to a file:

```bash
echo "$(git diff) Generate a conventional commit name for the current git changes" | qivryn -p > commit-message.txt
```

## How to Configure Qivryn CLI

`qivryn` uses [`config.yaml`](/reference), the exact same configuration file as Qivryn. This means that you can use your existing local configuration.

To switch between configurations, you can use the `/config` slash command in `qivryn`, or you can start it with the `--config` flag (e.g. `qivryn --config qivryn/default-cli-config` or `qivryn --config ~/.qivryn/config.yaml`).

### How to Add Custom Models

Learn how to add custom models [here](/customize/overview). Then, you can use the `/model` slash command to switch between them in `qivryn`.

### How to Configure Rules

`qivryn` supports [rules](/customize/deep-dives/rules) in the same way as the Qivryn IDE extensions. You can also use the `--rule` flag to manually include a rule. For example, `qivryn --rule nate/spanish` will tell `qivryn` to always speak in Spanish.

### How to Configure Tools

`qivryn` supports MCP tools, which can be configured in the [same way](/customize/deep-dives/mcp) as with the Qivryn IDE extensions.

#### How to Set Tool Permissions

`qivryn` includes a tool permission system to make sure you approve of the agent's actions. It will begin with minimal permissions but as you approve tool calls, it will add policies to `~/.qivryn/permissions.yaml` to remember your preferences.

If you want to explicitly allow or deny tools for a single session, you can use the command line flags `--allow`, `--ask`, and `--exclude`. For example:

```bash
# Always allow the Write tool
qivryn --allow Write()

# Always ask before running curl
qivryn --ask Bash(curl*)

# Never use the Fetch tool
qivryn --exclude Fetch
```

## Troubleshooting

Run `qivryn` with the `--verbose` flag to see more detailed logs. These will be output to `~/.qivryn/logs/qivryn.log`.

If you have feedback on the beta, please [file a GitHub issue](https://github.com/atnumridha/qivryn/issues).
